## 推荐组件边界

- `*Config : ScriptableObject`：稳定配置、曲线、表面映射、掉落池、升级表。
- `*Controller : MonoBehaviour`：输入、动画、物理、生命周期接入。
- `*RuntimeState`：纯 C# 状态，可测试，不依赖 Unity API。
- `*View` 或 `*Feedback`：VFX、音频、相机、UI 反馈。
- `*Pool`：对象池，隐藏实例化和回收细节。

## 输入宽容类

适用：土狼时间、输入缓冲、冲刺容差、连招缓冲、取消窗口。

实现要点：

- 在 `Update` 采集输入时间戳，在 `FixedUpdate` 或动作状态机中消费。
- 用秒作为配置单位，避免用帧数绑定帧率。
- 每次成功消费后清空对应缓冲，防止一次输入触发多次。
- 记录 debug 状态，便于 QA 看见剩余时间窗。

验收要点：

- 边界前 1 帧/边界后 1 帧都要测。
- 低帧率下时间窗不能明显变短。
- 同时输入多个动作时优先级必须明确。

## 战斗手感类

适用：命中暂停、相机震动、连招、后坐力、武器重量、命中确认。

实现要点：

- 把伤害判定、动画事件、反馈播放拆开，避免反馈失败影响伤害。
- Hit Pause 使用集中服务控制时间缩放或局部暂停，避免永久卡住。
- 相机震动用强度、频率、衰减曲线，而不是单个 magic number。
- 连招窗口用动画归一化时间或状态机事件配置。

验收要点：

- 连续命中不会堆叠到不可控暂停。
- 暂停不破坏 UI、冷却、异步加载或网络状态。
- 反馈强度随伤害/稀有度/武器重量分层。

## Roguelike 构筑类

适用：道具池、叠加效果、协同组合、局外成长、商店、保底。

实现要点：

- 用 ScriptableObject 定义物品、效果、稀有度、标签、互斥关系。
- 运行时把“数值 modifier”和“行为 modifier”分开。
- 叠加规则必须声明：加法、乘法、最大值、覆盖、独立触发。
- 随机池要支持权重、排除、保底、固定 seed。

验收要点：

- 同一 seed 结果可复现。
- 极端叠加不会溢出、负数、无限循环或锁死。
- 新手前 5 分钟能看到至少一个明确构筑方向。

## 性能敏感类

适用：对象池、空间查询、批处理、LOD、粒子、UI 刷新。

实现要点：

- 热路径用 `NonAlloc` API 或缓存数组。
- 大量对象分帧处理，避免同一帧全量刷新。
- UI 用脏标记刷新，不在 `Update` 重建文本和布局。
- 粒子、音效和屏幕效果提供质量档位。

验收要点：

- Profiler 中 GC Alloc 接近 0B/frame。
- 峰值对象数达到目标规模时帧时间仍在预算内。
- 对象回收后状态完全重置，无残留特效、协程或事件订阅。
