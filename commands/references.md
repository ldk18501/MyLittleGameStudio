# Command: references

## 目的

收集并分析参考游戏、图片、风格、避让项和约束。

## 主负责人

Producer

## 支持角色

- Creative Director
- Technical Artist
- UI 参考重要时加入 UI/UX Developer

## 写入

- project `design/references.md`
- project `design/reference-analysis.md`

## 流程

1. 从项目状态解析活动项目。
2. 通过自然对话或用户提供的文件/链接收集参考。
3. 在 `design/references.md` 记录原始参考。
4. 分析：
   - 借鉴什么
   - 避免什么
   - 对玩家幻想的影响
   - 对视觉可读性的影响
   - 对 UI/control 的影响
   - production risk
5. 写入 `design/reference-analysis.md`。
6. 把项目状态的 next action 更新为 `concept`。

## 自动化

- High：直接分析并呈现综合结论。
- Medium：分析前确认参考清单。
- Low：确认重要的 borrow/avoid 解读。
