# Command: test

## Purpose

Run, define, or summarize verification for the current task or project phase.

## Lead

QA Lead

## Supporting Agents

- Gameplay Developer
- Unity Architect
- UI/UX Developer

## Procedure

1. Determine target:
   - current task
   - Unity compile
   - smoke test
   - balance simulation
   - UI walkthrough
   - build preflight
2. Run available checks when the environment supports them.
3. If checks cannot be run, create a manual verification plan.
4. Record results:
   - command/check
   - pass/fail
   - evidence path
   - issues found
   - next fix if needed

## Completion

- Verification result or test plan exists.
